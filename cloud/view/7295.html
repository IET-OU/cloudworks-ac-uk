Demoing embeds, part 6 - Arbitrary files & documents

Nick Freear
14 January 2013

On 14 January 2013 we added functionality to Cloudworks to enable arbitrary files and documents from a list of domains to be embedded. (This includes CompendiumLD maps in SVG format.)

Note, this functionality is currently in alpha - there may be bugs, and behaviour is subject to change.

Supported domains:

  • dl.Dropbox.com -- Free/ paid storage on the Internet, may only work for files uploaded before January 2013 (possible change in Dropbox behaviour);
  • UbuntuOne.com -- Free/ paid storage on the Internet;
  • upload.Wikimedia.org -- Wikimedia Commons, part of the Wikipedia family, contains openly licensed image and multimedia;
  • OpenClipArt.org -- Public domain SVG and PNG images;
  • www.open.edu/openlearn --
  • www.open.ac.uk/blogs -- For example Andrew Brasher's blog;
  • Labspace.open.ac.uk -- 
  • CompendiumLD.open.ac.uk --
  • ORO.open.ac.uk -- PDF documents attached to research paper and book entries,
Supported file types and extensions - this is a subset of the >types supported by Google Drive Viewer:
  • pdf,
  • doc, docx, xls, xlsx, ppt, pptx,
  • zip, rar
  • svg, png
  • txt

 Notes:

  • Where the URL does not contain a recognizable file-name and extension (ie. on Ubuntu one storage), you will need to append a "name=.." parameter to the URL, eg. http://ubuntuone.com/5bfulWfbf1RQ225utYWVHe#!name=my-map.svg
  • You can optionally specify a width and height in pixels for the embed,
  • Flags and parameters can be combined in any order, eg. http://ubuntuone.com/5bfulWfbf1RQ225utYWVHe#!name=task-times-v1.svg&CompendiumLD=1&width=345&height=483
  • Except for CompendiumLD SVG maps, files are embedded using Google Docs Viewer - thank you Google! As with all embeds, we make no claims on the software and technology developed by 3rd party sites, and we attribute the 3rd party services where possible.
