OK - quick comment on my concerns around the context driver (I need hardly add that these are personal views and not the views of the UKOER programme).

Seeing the context that materials were originally used in as a driver for reuse makes a number of suppositions that need to be critiqued.

1. It presupposes we are dealing with "large" objects that could be said to contain pedagogic intent. Much OER is of a higher granularity, and can be seen as pure resource. An example of this latter would be something like an annotated diagram of the heart, or a description of the TCP/IP stack. The context that the image or description was originally used in is uninteresting - and you could imagine these materials used in a variety of contexts.

 2. It presupposes that academics will deal with "labelled" OER differently to the way that the population more generally deals with information on the web. In Broder's 2002 taxonomy (http://www.sigir.org/forum/F2002/broder.pdf), searching for OER is an "informational" query - with users seeking either "the best" source of that information (which is without any context) or a "hub" (which is a portal or repository containing a number of sources of the information required - where context MAY be one of the criterie used for selection). We could argue about whether those searching for OER are looking for "TCP/IP explanation" or "TCP/IP learning resources" - my instinct would be the former, though the latter would more likely present us with a "hub". More research is needed.

3. It presupposes that learning contexts are reproducable, and that a context in one learning environment would require a similar approach to the "same" context in another environment.

4. It presupposes that academic staff would prefer to use a pre-designed learning activity rather than use open materials to support their own design. There is a lot of literature around the ideas of sharing learning designs but this has yet to become mainstream. The findings of the JISC RePRODUCE programme back this assumption up, one project noting  that "“unless there are strong pedagogical reasons to incorporate material in a course in its entirety it is usually preferable to link to or embed it”. (http://www.jisc.ac.uk/whatwedo/programmes/elearningcapital/reproduce)

If we make those four suppositions, it is valid to argue that data regarding the context of materials is important. Otherwise, I would (personally) have problems with the issue.
