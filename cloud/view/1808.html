This was quite challenging. The collaborative patterns were fairly straightforward to grasp, esp. with Yannis' clear description. But Social work was a domain with which we weren't familiar. So, instead of using the handout, we based a design around our shared knowledge of creation of social work care packages for families with disabled children. We used the pyramid approach, but each of the 3 levels was implemented using one of the other approaches. First level was brainstorming issues, next level up was part of the jigsaw pattern in which learners specialised in small groups to clarify and resolve specific issues. Next level up they got together again into two groups, with representatives from each jigsaw group, then finally they peer-assessed each other's care package.

Not sure if this was exactly the right sort of approach, but it seemed to make sense to us.

Alex and Gill

One of the important elements of this design is the fact that it is technology neutral. It can be conducted equally well F2F or, if distance or time is an issue, synchonously or asynchonously using technologies. Web 2 technologies used in either case as repository for artefacts and shared knowledge.
