Nicole's review of e-Design Template and Web Collage

Cloud created by:

Review on W8/9 Activity 2a

e-Design Template >

Readability:
Very readable: time flow, increasing interaction, increasing level of knowledge, increasing diversity of assessment, from individual acquisition to collaborative participation.

Expressiveness:
A representation, a schema which I can understand on one sight and get to know the details by looking at the details. Very good to me: structured, not crowded. The Starter picture for an overview,  the macrolevel, then the table for the microlevel: detailed methodology.

Utility (communicating important  aspects of design):
Yes, it shows the procedural flow of the activity. Using different colour. Using arrow symbols to show the 'increasing' message.

 

Web Collage >

Readability:
There is a lot going on in just one picture...I would prefer six pictures (for each activity) the size of the one given, so that I can read it properly, then trying to understand what the tool tries to abstract.

Expressiveness:
It looks very complicated, though I am thinking it is a processual flow from top to bottom where the more complex activities are 'spelled out' in bigger picture representation, like close-up or zooming in.

Utility (communicating important  aspects of design):
It shows clearly that there are certain activities which come before others and that the methodology being used vary. I would not understand it, haven't I seen the original first.

 

In consideration of these 2 perspectives of learning design into my own documented Activity in 1b:

I like the e-Design Template a lot and can imagine myself using this format. I like the structuring nature of it. Start with a picture to layout the framework and the Learning Outcomes, show what the learner's progress is and how you assess it while being en route.

I cannot see where the Web Collage would assist me. I worked as a Project Manager and call myself literate in reading project plans. But this is not a learning plan in itself, it adds up, what methodology is in use..., no, not yet a tool I would consider.
