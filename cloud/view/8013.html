Testing and Data Collection Techniques

Given the type of course and desired testing objectives, the most appropriate data collection techniques would be:

1. Test ease of navigation and location of course resources: program monitoring / recording of learner actions

2. Test appeal of different components of course design: learners answers to specific design questions

3. Test effectiveness of video segments: notes / recording of pair discussions

Using these three data collection techniques in a coordinated fashion would provide a more holistic view of the course.

Prototyping Design Plan

The planned course has several key elements that must be included in the course prototype:

1) The course navigation and layout

2) An interactive course timeline and time requirements that reflects progress made and amoutn of time to complete. The student would earn / unlock cool tools, resources, or rewards (tbd) as they make progress.

3) Embedded media content (videos)

4) Interactive dialog tools (chats, forums, etc)

5) Self-assessment surveys and reports

6) Challenges that results in the awards based on leaderboards

7) Fillable forms for real-time case study analysis

Components 1-4 are easily reflected in any prototyping tool as they can have static layouts with prepopulated information used to illustrate any charts or graphs. However, components 5-7 require more dynamic prototyping features. There has to be the ability to link the prototype to live sites (e.g. survey monkey) and to allow the testers to complete a few of the challenges. For this reason, I would likely use a combination of tools.

Balsamiq can be used to provide all the visual prototyping but then the completed work would be pulled into a tool that would allow hotlinks to the sites were the self-assessments, surveys, and challenges were housed. Further evaluation of Balsamiq compatible tools is required to determine exactly which tool would be used.   

Extra content

Since video is so critical to the course, I decided to focus the observation on captivating and engaging videos. Specifically, I wanted to identify those attributes that aid in attentiveness and learning acquisition. For this observation, I used a supply chain mgmt conference that I was already participating in; this way, the participants did not know that an observation was occurring. Three videos were used of differing lengths. All video were ones that were professionally done and made available via a youtube channel. The resulting discussion / notes: 1. Personal stories are more captivating but only if its at the right level of detail. 2. Need to be able to see the immediate application 3. Use animation but don't overdo 4. Message in simple every dat terms; don't use technical jargon. It's an immediate turnoff. 5. Repeat key concepts but do so in different, creative ways that respect their intelligence 6. Too many really short videos is overwhelming though it makes for easier loading and downloading. 7. Seeing the full video series as an easily navigable toc is important (e.g. 1 of 6) What was left open was the length at which attention is lost regardless of the quality of the quality of the video.
